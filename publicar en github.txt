? Opciones para publicar manuales en GitHub
1. Publicar tus manuales como archivos dentro de un repositorio

Puedes crear un repositorio y subir tus manuales en formatos como:

.md (Markdown) ? el más recomendado

.pdf

.docx

.html

Los usuarios podrán leerlos directamente desde GitHub o descargarlos.

2. Crear un sitio web con GitHub Pages

Si quieres que tu manual tenga forma de página web, GitHub ofrece GitHub Pages gratis.

Con GitHub Pages puedes:

Convertir tus archivos Markdown en páginas web.

Usar plantillas (por ejemplo: Jekyll, MkDocs, Docusaurus).

Tener una URL pública como:
https://tuusuario.github.io/tu-manual

Ideal para manuales largos y organizados.

3. Publicar manuales en forma de Wiki

Cada repositorio puede tener una Wiki, que permite:

Crear capítulos y secciones.

Editar con interfaz sencilla.

Enlazar páginas.

Es cómodo para documentación técnica o manuales de procedimientos.

?? ¿Necesitas permiso para publicar?

Solo debes asegurarte de que:

El contenido es tuyo o tienes derecho a publicarlo.

No contiene información privada o confidencial.

Por lo demás, GitHub no pone restricciones.

Add file ? Create new file

?? 3. Crear GitHub Pages para tu manual

?? Configurar GitHub Pages (simple)

En tu repositorio, ve a Settings.

Deploy from a branch

Branch: main

Folder: /root (o /docs si usas esa carpeta)

https://tuusuario.github.io/manual-de-usuario

?? Truco: Menú lateral dinámico
1. Front matter en cada archivo

En la parte superior de cada archivo .md (front matter), asegúrate de tener al menos esto:

---
title: Nombre de la sección
nav_order: 2  # Cambia según el orden
published: true  # <--- Esto hace que se muestre en el menú
---


published: true ? La sección aparecerá en el menú lateral.

Si no quieres que aparezca aún, pon published: false.

Ejemplo para un archivo vacío que todavía no vas a usar:

---
title: Casos de estudio
nav_order: 7
published: false
---


Cuando quieras activarlo, solo cambia a true.

2. Configuración en _config.yml

Asegúrate de tener esta línea (Just the Docs lo soporta por defecto):

# Mostrar solo páginas publicadas en el menú
hide_unpublished_pages: true


Si tu _config.yml todavía no tiene esa opción, agrégala. Esto hace que cualquier archivo con published: false no aparezca en el menú lateral.

3. Beneficios

Mantienes el manual limpio mientras construyes contenido.

Las secciones incompletas no confunden a los usuarios.

Fácil de activar cuando termines de redactar cada sección.

Si quieres, puedo modificar tu estructura de repositorio vacía para que todos los archivos .md tengan published: false por defecto, y solo los que quieras mostrar aparezcan automáticamente en el menú. Esto te da un inicio muy limpio y profesional.

?? Opciones para GitHub Pages privado

GitHub Pages no permite sitios privados “nativos” para repositorios públicos. Pero podemos usar autenticación básica o servicios externos:

Opción 1: Usar GitHub + Jekyll localmente (más técnico)

Mantienes el repositorio privado.

Los usuarios autorizados clonan el repositorio:

git clone https://github.com/TU_USUARIO/manual-ceu-andalucia.git


Instalan Jekyll y ejecutan el sitio local:

bundle exec jekyll serve


Acceden desde su navegador a http://localhost:4000.

? Ventaja: nadie más puede acceder, solo quienes tengan acceso al repositorio y sepan cómo usar Jekyll.
? Desventaja: requiere instalación y conocimientos técnicos mínimos.

Opción 2: Publicar en un servicio externo con contraseña

Genera el sitio estático (Jekyll genera archivos HTML en _site/).

Súbelo a un servicio que permita proteger con contraseña:

Netlify (gratis) ? permite añadir autenticación básica

Vercel ? también se puede proteger con contraseña

GitLab Pages ? permite controlar acceso si usas GitLab privado

? Ventaja: tus usuarios solo necesitan un navegador y contraseña.
? Desventaja: hay un paso extra de subir los archivos a otro servicio.

Opción 3: Compartir solo archivos PDF/Markdown

Convierte tu manual en PDF o ZIP con todos los archivos .md.

Comparte con tus docentes o estudiantes mediante correo o Google Drive/OneDrive.

? Ventaja: muy simple y seguro.
? Desventaja: pierdes la interactividad de GitHub Pages.

?? Mi recomendación profesional para un manual privado de clases:

Mantén el repositorio privado en GitHub.

Invita solo a docentes y estudiantes como colaboradores.

Usa Opción 2 si quieres que puedan ver el manual en navegador sin instalar nada.
